The source data contains 753,000 rows of data about different stock data. This is approaching the limit of a workable excel sheet. The columns of the dataset are: Toggle Columns

Python

This project is seperated into two parts: PyBank & PyPoll

PyBank

In here we utilize Python's with open to read a CSV file. After reading in the data, the script will parse through and locate values within the file. The final outcome gets the difference from month to month, find the greatest difference in between months, and the smallest difference between months. If I were to go back and do this project again, I would definitely utililze Pandas, as well as Pythons min() and max() functions.

PyPoll

In here we once again utilize Python's with open to read a CSV file. After reading in the data, the script will parse through and locate values within the file. In particular, we are looking for the unique voter registration number, candidate name, and county they are voting from. The final outcome is what a polling site would do, tally up the total votes, analyze the percentage of votes from the total population, and declare a winner. If I were to go back and do this project again, I would definitely utililze Pandas, as well as Pythons min() and max() functions. Also, I would optimize for the program to run more than once... As is stands, it runs once and creates the folders and file locations without checking if they exist.

This is the Pymaceuticals challenge. This project reads in data from two CSV files which eventtaully refer to each other. To view columns from the metadata set:

Toggle metadata Columns:

  • Mouse ID - str
  • Drug Regimen - Cancer treatment company name - str
  • Sex - Male/Female - str
  • Age_months - int
  • Weight (g) - grams - int
  • Price - double

Toggle study result dataset Columns:

  • Mouse ID - str
  • Timepoint - Time taken from lab i nminutes - int
  • Tumor Volume (mm3) - Size of the tumor - int
  • Metastatic Sites - int

The project utilizes ETL to perform its analysis. The exctract phase was quite simple as the data was provided to me in a comma delimitted format. The Transform phase was not the same. To ensure the data is pure, I looked at the study results to ensure there is no duplicate data for any of the subjects. Unfortunately, there was some flawed data and had to be purged to ensure consistent data for the other subjects. Finally the load phase was to return the data back into a CSV file. Quite simple while utilizing Pandas.

In the Jupyter notebook, the analysis begins with cleaning the data. Next, we take a look at the Mean, Median, Variance, Standard Deviation, and Standard error of the Mean for the Tumor Volumes. Pandas makes this process smooth and easy. After identifying the data to look at, we are able to display this information in a clean DataFrame.

Folowing the beginning analysis, Matplotlib allows us to convert DataFrames into charts to display the information. The charts generated are: Bar Charts, Pie Charts, Box and Whisker Charts, Line Charts, and Scatter Charts.

This project utilizes OpenWeatherMap API to extract data about cities around the world. When creating the request, the script randomly generates an integer for latitude and longitude, then associate the nearest city to the values. The information I was looking for in the JSON response were: Toggle request information:

  • Name - str
  • Longitude - double
  • Latitude - double
  • Temperature - Farenheit - int
  • Humidity - int
  • Cloudiness - int
  • Wind Speed - Miles per Hour - double
After getting all of the data, I was able to run my analysis from the data.

For the analysis I wanted to understand how Temperature, Humidity, Cloudiness, and Wind Speed correlated to Latitude. To do so, I was able to make a chart for each independant value and find the line of correlation for each graph. Not satisfied with a global observation, I repeated the same steps while only looking at the Northern and Southern Hemisphere.

After performing the analysis, Google's Map API allowed me to associate the data I received from OpenWeatherMap API. I filtered the data to look at locations which had ideal vacation weather. If I were to go back and do this again, I would want to request data from past years about the month I would want to visit. I would identify the most optimal vacation by how the city behaved in previous years in hope to find the best month to go. After filtering, Google's API helped me find the hotel nearest to the center of the cities with the most ideal vacation weather.

Big Data

Here I utilize Amazon's, Amazon SageMaker, service to create a Virtual Environment to analyze a huge amount of data. In the repository, there are two Jupyter notebooks which perform the same tasks. Both notebooks will use a link to request the data from an S3 bucket, Load the data into a pyspark object, analyze the information with PySpark, and upload the data to a database with AWS. The data exists in the public S3 bucket. Loading data into the pyspark can be seen in the notebook. Analysis of the data includes converting PySpark DataFrames into Pandas DataFrames. Uploading the data is done with Amazon's offical SDK: boto3. The data is housed on a personal AWS account.
